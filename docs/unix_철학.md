# Unix 철학

## 1. 한 가지 일을 잘하라 (Do one thing well)

"Write programs that do one thing and do it well."

- 각 프로그램은 단일 기능에 집중해야 합니다.
- 예: ls(파일 목록), cat(파일 출력), grep(패턴 검색)

## 2. 프로그램이 협력하도록 설계하라 (Programs Work Together)

"Write programs to work together."

- 작은 도구들을 파이프(|) 로 연결하여 복잡한 작업 수행
- 예: cat file.txt | grep "error" | wc -l

## 3. 텍스트 스트림을 범용 인터페이스로 사용하라

"Write programs to handle text streams, because that is a universal interface."

- 텍스트는 프로그램 간 데이터 교환의 공통 언어
- 사람이 읽을 수 있고, 디버깅이 쉬움

| 원칙                                   | 설명                                   |
| -------------------------------------- | -------------------------------------- |
| **단순함을 추구하라**                  | 복잡한 해결책보다 단순한 해결책을 선호 |
| **이식성을 중시하라**                  | 효율성보다 이식성(portability)을 우선  |
| **데이터를 텍스트로 저장하라**         | 바이너리보다 텍스트 파일 선호          |
| **소프트웨어 지렛대를 활용하라**       | 기존 도구를 재사용하여 생산성 향상     |
| **쉘 스크립트로 자동화하라**           | 반복 작업은 스크립트로 처리            |
| **과도한 UI를 피하라**                 | 사용자를 "사로잡는" 인터페이스 지양    |
| **모든 프로그램은 필터가 되어야 한다** | 입력을 받아 변환 후 출력하는 구조      |
| **일찍 실패하라**                      | 오류는 빨리, 명확하게 알려야 함        |
| **침묵의 규칙**                        | 할 말이 없으면 아무것도 출력하지 않음  |
