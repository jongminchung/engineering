# CNCF 프로젝트 기반 K8s 로드맵

## 목적

- CNCF 프로젝트를 직접 써보며 아키텍처 관점과 쿠버네티스 운영 이해를 쌓는다.
- CSP(클라우드 서비스 제공자)의 관리형 K8s 서비스까지 이어질 수 있는 경험을 만든다.

## 단계별 로드맵

### 1단계: 쿠버네티스 기본기 확립 (2-4주)

- 핵심 오브젝트: Pod, Deployment, Service, Gateway API, ConfigMap, Secret, Volume
- 네트워크/스토리지 개념: CNI, CSI, 서비스 디스커버리
- 목표: 워크로드가 어떻게 배포되고 통신하는지 설명 가능

### 2단계: CNCF 핵심 프로젝트 체험 (4-6주)

- 관측성: Prometheus + Grafana, Loki/Tempo 중 1개
- 배포/운영: Argo CD (GitOps)
- 서비스 메쉬: Istio 또는 Linkerd
- 목표: 운영 관점에서 구성 요소를 선택한 이유를 설명 가능

### 3단계: 아키텍처 관점 설계 연습 (3-5주)

- 레퍼런스 아키텍처 문서화 (예: 마이크로서비스 + 관측성 + 보안 + 배포 흐름)
- 장애 시나리오 설계 (노드 장애, 네트워크 문제, 롤백)
- 목표: 설계 의도와 트레이드오프를 글로 정리 가능

### 4단계: CSP 관리형 K8s 서비스 이해 (2-4주)

- GKE/EKS/AKS 중 1개 선택
- IAM, 네트워크, 스토리지, 로깅/모니터링 연동 방식 파악
- 목표: 관리형 K8s와 OSS K8s 차이를 설명 가능

### 5단계: 포트폴리오/실전 적용 (3-6주)

- 실제 앱 배포 + 관측성 + GitOps + 보안 정책 적용
- 장애 복구/확장/비용 최적화 시나리오 작성
- 목표: CSP에 K8s 서비스를 제공할 때 필요한 운영 요건 정리

## 추천 학습 순서

1. K8s 기본기 + 간단한 워크로드 배포
2. Prometheus/Grafana로 관측성 추가
3. Argo CD로 GitOps 배포 구축
4. Istio/Linkerd로 트래픽/보안 제어
5. CSP(K8s 관리형)로 동일 구조 재현

## 산출물 체크리스트

- K8s 핵심 오브젝트 설명 문서
- 관측성 대시보드 스크린샷/링크
- GitOps 배포 흐름 다이어그램
- 서비스 메쉬 적용 전/후 트래픽 정책 요약
- CSP 관리형 K8s 비교표 및 운영 차이 정리
