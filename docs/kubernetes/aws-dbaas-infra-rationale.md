# AWS DBaaS가 VM/전용 인프라 중심인 이유

## 결론

AWS의 관리형 DBaaS는 대부분 VM/전용 인프라 기반이다.
IOPS도 중요한 이유지만, 실제로는 성능 일관성, 격리, 운영 안정성, 스토리지 구조가 더 큰 결정 요인이다.

## 주요 이유

- 성능 예측 가능성: DB는 IO/CPU 편차에 민감해서 VM/전용 인프라가 지연과 처리량을 안정화하기 좋다.
- 스토리지 아키텍처 제어: Aurora처럼 컴퓨트/스토리지를 분리한 전용 스토리지 계층이 필요하다.
- 강한 격리와 SLA: 멀티테넌시 간섭을 최소화하고 성능 SLA를 보장하기 쉽다.
- 운영 자동화 단순화: 백업/복구/업그레이드/페일오버를 표준화된 VM 기반으로 안정적으로 자동화한다.
- 커널/네트워크 튜닝: DB는 OS, 네트워크, 파일시스템 튜닝이 중요한데 VM이 제어 범위가 넓다.
- 보안/규정 준수: 감사와 컴플라이언스 요건을 전용 인프라 모델로 맞추기 유리하다.

## IOPS는 어떤 위치인가

- IOPS는 핵심 요인 중 하나지만 단독 결정 요인은 아니다.
- 실제로는 IOPS를 포함한 "성능 일관성"과 "격리" 요구가 함께 작동한다.

## K8s 기반 DBaaS가 선택되는 경우

- 중소규모/내부 플랫폼에서 선언형 운영 자동화가 중요할 때
- 비용/운영 효율이 SLA보다 우선일 때
- 특정 벤더 오퍼레이터 기반으로 빠르게 서비스화해야 할 때
