# 영향력

다른 동료 또는 팀을 설득하고 좋은 영향을 주는 능력.

- 커뮤니케이션을 효율적이고 논리적으로 하는 능력.
- 긍적적인 관계 형성으로 업무 효율을 높이는 능력.

**영향력 파악 질문 예.**

- 업무를 위해 누군가를 설득한 경험.
- 같이 일하는 동료와의 좋은 관계 형성으로 프로젝트에 긍정적인 영향을 끼친 경험.
- 동료나 다른 팀을 설득해 프로세스를 개선해야 했던 경험.
- 다양한 이해 관계 속에서 주요 관계자들의 요구를 성공적으로 만족시킨 경험.

**좋지 못한 답변 특징.**

- 설득에 필요한 정량적인 데이터를 활용하지 못한 경험. (그러한 환경이 아니라면..?)
- 개인 관심사 중심으로 업무를 진행한 경험

## 사례 1. semantic-release

- `semantic-release`: semantic versioning을 자동으로 관리해주는 도구 사용을 도입.
- 해당 프로젝트를 사용하는 것을 단순히 하나의 프로젝트를 도입하는 것이 아니라 도입을 증진하는 방향으로 나아감.
  - 스케줄 CI를 통해 `badge`로 semantic-release로 관리되는 프로젝트임을 표현.
  - 사내 문서에서 어떻게 적용하면 되는지 설명을 남기고 이를 팀 내 공유 및 전파.
  - 점진적으로 수동적으로 CHANGELOG를 작업하는 방식에서 벗어나 conventional commit 문화와 함께 semantic-release 문화를 전파함.

## 사례 2. cicd cli

- `cicd cli` 반복되는 작업을 자동화하는 도구로 사용을 도입.
  - CI/CD 파이프라인에서 반복되는 작업을 자동화하여 효율성을 높임.
  - 팀 내 공유 및 전파를 통해 일관된 작업 흐름을 유지.
  - 점진적으로 기존의 수동 작업을 자동화하여 팀의 생산성을 향상시킴.
