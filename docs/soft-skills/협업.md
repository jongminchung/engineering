# 협업

개인의 목표가 아닌 공동의 목표를 달성하려 노력.

- 소통을 통해 의견을 공유하고 이해한다.
- 적극적으로 피드백을 주고 받는다.
- 역할과 책임을 분담하여 효율적인 작업을 수행한다.
- 문제 해결을 위해 다양한 관점을 고려한다.
- 협력적인 환경을 조성하여 팀의 성장을 지원한다.


## 사례 1. Open API Specification First

API 설계 단계에서 UI 팀과 협업하여 요구사항을 정의하고, API 설계를 진행한다.

이를 통해 개발과 UI 작업의 효율성을 높이고, 사용자 경험을 개선할 수 있다.

기존에는 API 개발과 UI팀들이 함께 리뷰하는 것이 아니라 API 개발 이후 UI 팀에게 전달하는 방식으로 진행되어, 개발과 UI 작업의 효율성이 떨어졌다.

이외에 API 스펙을 공유하는 과정에서 FE에서는 해당 스펙에 맞는 코드를 바로 작성하고 작업을 진행할 수 있어 팀의 생산성을 높일 수 있다.
